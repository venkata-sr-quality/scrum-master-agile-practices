# Retrospective Examples – Continuous Improvement

## Retrospective Purpose
Retrospectives help the team:
- Reflect honestly
- Learn from experience
- Improve incrementally

## Example 1: Start / Stop / Continue
- Start: What new practices should we try?
- Stop: What is wasting time or creating friction?
- Continue: What is working well?

## Example 2: What Went Well / What Didn’t / What We’ll Improve
Used after stressful or complex sprints.

## Example 3: Root Cause Focus
Identify a recurring issue and ask “Why?” multiple times.

## Action Items
- Limit to 1–3 improvement actions
- Assign ownership
- Review progress in next retrospective

## Scrum Master Insight
If retrospectives don’t produce change,
the team stops believing in Agile.
